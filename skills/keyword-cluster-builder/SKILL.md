# Keyword Cluster Builder

A keyword list is not a content plan. Publishing one thin page per keyword cannibalizes your own rankings and wastes crawl budget. The job is to collapse hundreds of queries into a small number of clusters, each earning ONE page, organized so internal authority compounds.

## Cluster by Intent and Result Type, Not String Similarity

Group queries that Google would answer with the same page. "best running shoes" and "top running shoes 2026" share a SERP; "running shoe size guide" does not. The fastest signal: if two queries return largely the same top-10 URLs, they belong on one page. Never split a cluster just because the wording differs.

## Assign Search Intent First

Label every cluster informational, commercial, transactional, or navigational. Informational clusters become guides and hubs; commercial become comparisons and "best X" lists; transactional become product/category/landing pages. Mixing intents on one URL satisfies none of them.

## Build the Pillar/Cluster Architecture

Each topic gets one broad pillar page targeting the head term, surrounded by cluster pages targeting long-tail subtopics. Every cluster page links up to the pillar with descriptive anchor text; the pillar links down to each cluster. This topical mesh is what signals authority on the whole subject, not just one keyword.

## Pick the Primary Keyword and Map the Rest

For each cluster choose one primary keyword (highest relevant volume the page can realistically win) and list secondary keywords it should also satisfy in subheads and body. Note difficulty honestly: a domain with little authority should target lower-difficulty long-tail clusters first and build up.

## Output a Prioritized Build Order

Sequence clusters by opportunity = relevance to revenue x winnability / current coverage. Flag overlaps where two existing pages compete for one cluster (consolidate them). Mark gaps where a cluster has demand but no page.

## What to Skip

Do not cluster branded/navigational queries you cannot rank for, near-zero-volume singletons, or queries whose SERP is pure ads/maps with no organic opportunity. Rankings take months; do not promise timelines, and never recommend doorway pages or spun variants to cover micro-keywords.
