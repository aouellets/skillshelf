# Variant Copy Scaler

When a product ships in twelve colors and four sizes, copying the same paragraph 48 times creates thin, duplicate content that search engines discount and shoppers find unhelpful. The job is to keep one strong master narrative while making each variant page distinct enough to rank and specific enough to answer "is THIS the one I want." Ask for the master description and the full variant matrix first.

## Separate Shared Core From Variant Layer
Write the brand story, materials, and core benefit once as a shared core. Then write a thin variant layer that changes per option: the color's mood and what it pairs with, the size's fit and who it suits, the bundle's value math. The rendered page is core plus layer, so 80% is consistent and 20% is genuinely unique to that variant. Never just swap a single color word and call it new copy.

## Make Each Variant Earn Its Own Words
For color, describe the actual hue honestly ("warm oatmeal, not bright white") and a styling cue. For size, state who it fits and the use case ("the 12oz suits a single espresso drinker; the 20oz is for all-day desk sipping"). For bundles, show the per-unit value and the occasion (gifting, stocking up). This specificity both converts and gives search engines distinct text.

## Avoid Duplicate-Content Penalties
If the platform creates a separate indexable URL per variant, ensure each has materially different copy or set canonical tags to the parent product. Decide deliberately: rank variants individually (needs unique copy) or consolidate to the parent (use canonicals). Don't generate near-identical pages that compete with each other and dilute authority. Flag the canonical decision explicitly in your output.

## Keep Voice and Structure Locked
Use the same section order, bullet count, and tone across every variant so the catalog feels coherent. Build a template with slots, then fill the slots per variant. This is what makes the output scale cleanly to dozens of SKUs without drift.

## Honesty Across Variants
Don't claim a benefit for one color that isn't true (a light fabric isn't "slimming" just because it's black). Keep care, warranty, and shipping identical unless they genuinely differ by variant.

## When to Push Back
If the variant matrix is huge and undifferentiated (e.g., 30 near-identical shades), recommend consolidating to a parent page with a swatch selector rather than spinning thin pages. Ask before generating something that will hurt SEO.
